Documentation

This page describes the live components, endpoints, and security posture of Automation Central.

System Overview

Automation Central is a Creator OS for routing ideas, content, and requests into structured automations and agents.

  • Frontend: Cloudflare Pages
  • API: Cloudflare Worker (ac-api)
  • Intake Logging: Google Sheets
  • Automation Runtime: n8n (gated)

Live Endpoints

Public

GET https://api.automationcentral.online/health

Health check for the API.

Public (origin-restricted)

POST https://api.automationcentral.online/intake-public

Used by the website intake form.

Private (keyed)

POST https://api.automationcentral.online/intake

Used by automations and agents with x-ac-key.

Modules

  • ARIA Intake – normalize and route requests (/modules/aria)
  • Publisher Ops – scheduling and publishing (/modules/publisher)
  • Competitive Intel – monitoring and reporting (/modules/intel)

Security

  • HTTPS enforced via Cloudflare
  • CORS restricted to canonical origin
  • Private intake endpoint requires API key
  • n8n is gated behind Cloudflare Access

Status

  • API: Operational
  • Intake: Operational
  • Modules: Live / Planned
