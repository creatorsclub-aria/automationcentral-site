How it works

Automation Central turns your goal into a working system. You do not need to be technical to start.

The simple version

1) Tell us the outcome

Use Intake and describe what you want to happen, where, and how often.

2) We route it

We match your request to the right module (ARIA, Publisher, Intel).

3) You get the build

Functional workflow plus setup steps and logging.

What you get
  • A working workflow (not theory)
  • Clear setup steps and sanity checks
  • Logging so you can see what ran and what failed
  • A clear next action (what to do after it runs)
What’s live right now
  • Website intake logs to Google Sheets
  • API is live and operational
  • n8n is protected behind Cloudflare Access
For builders (technical)

These are the live endpoints used by the system:

  • GET https://api.automationcentral.online/health
  • POST https://api.automationcentral.online/intake-public (origin-restricted, used by the website)
  • POST https://api.automationcentral.online/intake (keyed, used by automations and agents)

Modules:

  • /modules/aria routing and intake normalization
  • /modules/publisher publishing operations
  • /modules/intel competitive intelligence
