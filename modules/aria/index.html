ARIA: Intake & Routing

ARIA takes messy requests and turns them into clear build instructions. If you do not know what to ask for, start here.

Start with Intake Back to Modules How it works

What ARIA does

Clarify

Turns your goal into a clear description of what the system should do.

Route

Sends the request to the right build path (Publisher, Intel, or custom automation).

Deliver

You receive a working workflow with setup steps, logging, and a next action.

What you should include in your request

  • The outcome you want (in one sentence)
  • Where it should happen (platforms/tools)
  • How often it should run
  • What counts as “success” (example: logged to a sheet, email sent, post published)

Good example requests

  • “When someone fills out my form, save their info, send a confirmation email, and notify me in Telegram.”
  • “Schedule my content weekly and log every post to a Google Sheet with a weekly summary email.”
  • “Track three competitors weekly and write a summary to a sheet every Sunday.”
